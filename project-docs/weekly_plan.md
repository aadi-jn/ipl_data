# IPL Analytics — 9-Week Plan

**Start:** Mar 31, 2026 | **End:** ~Jun 1, 2026 (IPL season)

Priorities will shift weekly as we learn what works. Each week has 4-6 concrete items — no more. The rest is stretch.

---

## Week 1 — Data & First Impressions (Mar 31 – Apr 6)

1. Ball-by-ball data processed and queryable in Athena
2. Derived summary tables: player career stats, team season records, venue stats
3. Redesign frontend — move from SQL editor to something people want to spend time on
4. Pre-match analysis cards using historical data (H2H, venue, form, toss)
5. First LinkedIn post: announcing the project

## Week 2 — Pre-Match Engine & Visual Identity (Apr 7 – Apr 13)

1. Pre-match analysis fully working for upcoming IPL 2026 matches
2. 2-3 interactive charts (team timelines, toss impact, venue profiles)
3. Auto-insights prototype — LLM generates insights from pre-computed queries (local/free model)
4. Mobile-friendly layout
5. LinkedIn: 2 outcome posts + 1-2 behind-the-scenes posts

## Week 3 — Ball-by-Ball Analytics & Depth (Apr 14 – Apr 20)

1. Deep ball-by-ball analytics: powerplay/middle/death splits, phase-wise performance
2. Player matchup and performance pages
3. 2-3 more visualizations (death overs specialists, powerplay evolution, clutch performers)
4. Auto-insights refined and visible on the site
5. LinkedIn: insight deep-dives with chart screenshots

## Week 4 — AI-Enabled Features Begin (Apr 21 – Apr 27)

1. Auto-insights engine running daily for live IPL 2026 matches
2. Post-match analysis cards (turning points, player ratings, AI-generated summaries)
3. Start integrating real-time data source
4. Additional tabs taking shape (team analysis, player leaderboards)
5. LinkedIn: AI insights reveal — "what AI sees that dashboards can't"

## Week 5 — Real-Time & Live Match Features (Apr 28 – May 4)

1. Real-time data pipeline operational
2. Live match context: par scores, win probability, phase comparisons
3. Points table with playoff math and NRR scenarios
4. Polish existing features based on user feedback
5. LinkedIn: live match analysis posts, tool demos

## Week 6 — NL Queries (Testing) & MCP (May 5 – May 11)

1. NL query prototype — internal testing only, not public
2. MCP server prototype for programmatic data access
3. More use cases with existing data (season narratives, record watch)
4. Guardrails and metric definitions for NL queries
5. LinkedIn: collaboration invites, behind-the-scenes on NL/AI challenges

## Week 7 — NL Queries (Public) & Richer Content (May 12 – May 18)

1. NL queries enabled publicly (if tested and ready — otherwise stay in testing)
2. Text-based data sources: commentary, pre-match analysis ingestion
3. Animated match summaries or richer post-match content
4. Improve real-time features based on what's working
5. LinkedIn: NL query demos, user stories

## Week 8 — Engagement & Connectors (May 19 – May 25)

1. MCP servers live — Slack, Claude access to IPL data
2. More engaging features: shareable stat cards, social content
3. Player and team season summaries (IPL nearing end)
4. NRR scenarios, points table race, playoff projections
5. LinkedIn: connector demos, "access IPL data from your Slack"

## Week 9 — Season Wrap & Polish (May 26 – Jun 1)

1. End-of-season content: player performances over time, team summaries, season narrative
2. Polish everything — speed, mobile, visual consistency
3. About page / project story for new visitors
4. Retrospective: what worked, what to keep running post-IPL
5. LinkedIn: "what I learned building this" wrap-up series
