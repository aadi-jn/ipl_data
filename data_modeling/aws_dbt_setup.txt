  ---                                                                                                                                                                                   
  What you need to do for dbt setup                                                                                                                                                     
                                                                                                                                                                                        
  No dbt Cloud account needed. You're using dbt CLI — it's just a Python package. Everything runs locally and in GitHub Actions.
                                                                                                                                                                                        
  1. Install locally
  pip install dbt-core dbt-athena-community                                                                                                                                             
                                           
  2. Create ~/.dbt/profiles.yml (local only, never commit this)
  ipl_dbt:                                                                                                                                                                              
    target: dev
    outputs:                                                                                                                                                                            
      dev:        
        type: athena
        region_name: ap-south-1
        s3_staging_dir: s3://ipl-athena-results-814871720600/dbt/
        s3_data_dir: s3://ipl-processed-data-814871720600/dbt/                                                                                                                          
        database: ipl_cricket        # Glue database name     
        schema: dbt_dev              # Glue schema prefix for dev                                                                                                                       
        work_group: ipl-workgroup                                
        aws_profile_name: default    # your local AWS CLI profile                                                                                                                       
      prod:                                                      
        type: athena                                                                                                                                                                    
        region_name: ap-south-1                                                                                                                                                         
        s3_staging_dir: s3://ipl-athena-results-814871720600/dbt/                                                                                                                       
        s3_data_dir: s3://ipl-processed-data-814871720600/dbt/                                                                                                                          
        database: ipl_cricket                                 
        schema: dbt_prod                                                                                                                                                                
        work_group: ipl-workgroup
        # prod uses IAM role from GitHub Actions env, no profile needed                                                                                                                 
                                                                       
  3. Verify AWS CLI is configured with a profile that has Athena + S3 + Glue access — you likely already have this since you've been running aws s3 cp commands.                        
                                                                                                                                                                                        
  4. In GitHub: go to Settings → Secrets → add AWS_ROLE_ARN (the IAM role you'll create for OIDC). That's the only secret needed — no access keys.                                      
                                                                                                                                                                                        
  That's the full setup. No paid accounts, no cloud services beyond what's already deployed. The dbt project itself lives in the repo under ipl_dbt/ and GitHub Actions handles prod
  runs for free.
